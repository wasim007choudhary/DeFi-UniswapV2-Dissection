output of swap i = input of swap i+1

and specifically what i+1 means.

Let’s break it down clearly.

Protocol context: Uniswap V2

1️⃣ What i Means

i simply represents the index of a swap step in the path.

Think of the swap path:

[WETH, DAI, MKR]

There are two swaps here:

Swap index	Trade
i = 0	WETH → DAI
i = 1	DAI → MKR

So i is just a counter for the swaps.

2️⃣ What i + 1 Means

i + 1 means the next swap in the sequence.

Example:

swap 0 → WETH → DAI
swap 1 → DAI → MKR

So:

output of swap 0 = input of swap 1
3️⃣ Example With Numbers

Swap 1 (i = 1)

1800 DAI → ? MKR

So:

input of swap 1 = output of swap 0

Which matches the rule:

output of swap i = input of swap i+1
4️⃣ Why This Notation Is Used

When writing algorithms or smart contracts, we use indices:

amounts[i]
amounts[i+1]

Example array:

amounts = [1 WETH, 1800 DAI, 8 MKR]

Meaning:

index	value
0	1 WETH
1	1800 DAI
2	8 MKR

The swap logic uses these positions to calculate each step.
